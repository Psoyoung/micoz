## 1. 제품 개요

### 1.1 비전
고급스러운 분위기 프리미엄 화장품 브랜드가 **오픈마켓 의존을 줄이고**, 자사몰에서 고객을 직접 확보·유지하는 D2C 커머스 플랫폼을 갖춘다.

### 1.2 해결할 문제
| # | 문제 | PRD가 제공하는 해법 |
|---|---|---|
| P1 | 오픈마켓 수수료·고객 데이터 미확보 | 자사 회원·주문·행동 데이터 자체 축적 |
| P2 | 브랜드 경험 통제 불가 | 자사 디자인의 카탈로그·스토리·상세 경험 제공 |
| P3 | 재구매 유도 수단 부재 | 회원 등급·포인트·쿠폰 기반 리텐션 체계 |
| P4 | 운영 데이터 분산 | 단일 백오피스에서 상품·주문·CS·반품·통계 통합 관리 |

### 1.3 제품 목표 (Product Goals)
- **G1** — 상품 탐색부터 결제까지 **3분 이내** 완료 가능한 커머스 전 주기 제공.
- **G2** — 회원 등급·포인트·쿠폰으로 **재구매율 상승** 기반 마련.
- **G3** — 관리자가 코드 배포 없이 상품·배너·주문·CS를 운영.
- **G4** — 동시 사용자 10,000명 환경에서 안정적(99.9%) 서비스.

## 4. 범위 (Scope)

### 4.1 In Scope (이번 릴리스)
- 사용자: 인증, 홈/브랜드, 상품 탐색·상세, 장바구니·찜, 주문·결제, 주문후속(배송추적·리뷰·반품/교환), 마이페이지(주문·반품·찜·리뷰·배송지·회원정보·문의).
- 관리자: 대시보드, 회원·카테고리·상품·주문·반품·문의 관리, 배너·배송 설정, 관리자 계정 관리.
- 플랫폼: JWT 인증, 균일 REST API, PostgreSQL 영속화, Docker Compose 로컬 환경.

### 4.2 Out of Scope (이번 미포함, W)
- 비회원 게스트 주문, 소셜 로그인.
- 메시지 브로커(Kafka 등)·검색엔진(Elasticsearch)·캐시(Redis) 도입.
- 관리자 세분화 권한(RBAC 다단계), 다국어 i18n 백엔드 처리.
- 미연결 화면 기능: `SalesView`(매출 통계 별도 메뉴), `SettingsBrand/Notify/Api` — §11에서 도입 결정 후 별도 범위.

## 5. 기능 요구사항 (Functional Requirements)

> 형식: Epic → 사용자 스토리 → 수용 기준(AC). 각 요구사항에 ID·우선순위 부여. AC는 검증 가능한 조건으로 기술.

### EPIC A. 인증 / 계정 (Auth)

**FR-AUTH-01 회원가입** · **M**
- 스토리: 방문자로서 약관에 동의하고 가입하여 회원 혜택을 받고 싶다.
- AC:
  1. 이용약관·개인정보(필수) 미동의 시 가입 차단, 마케팅(선택) 분리 수집.
  2. `user_id` 중복 시 `USER_DUPLICATED_ID` 반환(단, enumeration 방지 메시지 정책 준수).
  3. 비밀번호는 BCrypt(strength 12)로 저장, 평문 미보관.
  4. 동의 일시(`*_agree_date`)가 기록된다.

**FR-AUTH-02 로그인 / 토큰 발급** · **M**
- AC:
  1. 성공 시 access(30분) + refresh(14일) 발급.
  2. refresh는 SHA-256 해시로 `dat_refresh_token`에 저장, 발급 IP 기록.
  3. 실패 시 아이디 없음/비번 불일치를 **동일 응답**(`AUTH_INVALID_CREDENTIALS`)으로 반환.
  4. `last_login_date` 갱신.

**FR-AUTH-03 토큰 갱신(rotation)** · **M**
- AC:
  1. 유효 refresh 제출 시 새 access+refresh 발급, 이전 토큰 `revoked_yn='Y'`.
  2. 이미 revoked된 토큰 제출 시 해당 사용자 전체 토큰 무효화(재사용 탐지).
  3. 만료/위조 토큰은 `AUTH_TOKEN_EXPIRED`/거부.

**FR-AUTH-04 로그아웃** · **M** — refresh 토큰 revoke 처리.

**FR-AUTH-05 아이디/비밀번호 찾기** · **S**
- AC: 본인확인 후 아이디 안내/비밀번호 재설정. 존재 여부 노출 없는 응답.

### EPIC B. 상품 탐색 (Catalog)

**FR-PROD-01 상품 목록** · **M**
- AC:
  1. 카테고리(2단계) 필터·정렬·페이징(`page/size/sort`) 지원.
  2. `display_yn='Y'` & `use_yn='Y'` 상품만 노출.
  3. 대표 이미지·라벨(BEST/NEW 등)·가격 표시.

**FR-PROD-02 상품 상세** · **M**
- AC: 옵션(용량별 가격·재고), 이미지(MAIN/SUB/DETAIL), 성분·사용법, 리뷰 요약, 장바구니/찜 담기 진입 제공.

**FR-PROD-03 홈/브랜드 스토리** · **S**
- AC: 노출중(`display_yn='Y'`) 히어로 배너 슬라이드 정렬 표시, 추천/베스트 상품 노출, 브랜드 스토리 페이지 제공.

### EPIC C. 장바구니 / 찜 (Cart & Wishlist)

**FR-CART-01 장바구니 CRUD** · **M**
- AC:
  1. 로그인 사용자만 담기 가능. 옵션 필수 상품은 옵션 미선택 시 `CART_OPTION_REQUIRED`.
  2. 동일 상품+옵션 재담기 시 수량 합산.
  3. 수량 변경·삭제·선택 주문 지원.

**FR-FAV-01 찜 토글** · **S** — 상품 찜 추가/해제(`map_product_fav`), 마이페이지에서 목록 조회.

### EPIC D. 주문 / 결제 (Order & Payment)

**FR-ORDER-01 주문 생성(체크아웃)** · **M**
- AC:
  1. 배송지(저장된 배송지 선택/신규 입력), 쿠폰, 포인트 적용 가능.
  2. **금액은 서버에서 재계산**(상품합 − 쿠폰 − 포인트 + 배송비 = `final_amount`); 클라이언트 값 불신뢰. 불일치 시 `ORDER_AMOUNT_MISMATCH`.
  3. 주문 시점 상품명·옵션·단가를 `dat_order_item`에 **스냅샷** 저장.
  4. 무료배송 기준(`mst_shipping.free_shipping_min`)·도서산간 추가비 반영.
  5. 주문번호(`order_no`) 생성, 초기 상태 `PENDING`.

**FR-PAY-01 결제 처리** · **M**
- AC:
  1. PG 승인 연동 후 결과 검증, 성공 시 주문 `PAID`·결제 `PAID` 전이, `paid_date` 기록.
  2. 승인 실패 시 `PAY_APPROVAL_FAILED`, 주문 미확정(롤백).
  3. 카드번호는 마스킹(`card_no_masked`)만 저장.
  4. 포인트 적립 예정(`point_to_earn`) 산정, 사용 포인트 차감 이력 기록.
- 의존: PG 사업자 확정(§11).

**FR-ORDER-02 주문 완료/내역/상세** · **M**
- AC: 주문 완료 화면에 주문번호·금액 표시. 마이페이지에서 본인 주문만 목록·상세 조회(타인 주문 접근 차단).

### EPIC E. 주문 후속 (Fulfillment)

**FR-SHIP-01 배송 추적** · **S**
- AC: 주문 상세에서 운송장 번호·배송 상태(READY/SHIPPED/IN_TRANSIT/DELIVERED) 표시. 외부 택배사 API 연동은 사업자 확정 후(§11).

**FR-RET-01 취소/교환/반품 신청** · **M**
- AC:
  1. 사유 유형(단순변심/불량/오배송/기타) 선택, 대상 주문상품·수량 지정.
  2. 신청 시 `dat_return` 생성(상태 `REQUESTED`), 교환 시 대체 옵션 지정 가능.
  3. 반품 가능 기간/상태 검증, 위반 시 `RETURN_PERIOD_EXPIRED`/`ORDER_INVALID_STATUS`.

**FR-REV-01 리뷰 작성/조회** · **S**
- AC: 평점(1~5)·내용 필수, 이미지 첨부(URL) 가능. 구매자 한정 작성 정책은 §11 확정 대상.

### EPIC F. 마이페이지 (My Page)

**FR-MY-01 회원정보 조회/수정** · **M** — 정보 수정, 비밀번호 변경(변경 시 전체 refresh revoke).
**FR-MY-02 배송지 관리** · **M** — 등록/수정/삭제, 기본 배송지(`default_yn`) 설정.
**FR-MY-03 포인트·쿠폰 조회** · **S** — 보유 포인트 잔액·적립/사용 이력, 보유 쿠폰(상태별) 조회.
**FR-MY-04 1:1 문의** · **M** — 유형별 문의 등록, 비공개 설정, 내역·답변 조회.

## 7. 데이터 요구사항

- 영속 저장소: **PostgreSQL 15**, 스키마 `micoz_schema.sql`(26 테이블) — 임의 변경 금지, 변경은 Flyway 마이그레이션으로 관리.
- 소프트 딜리트: `use_yn='N'` 마킹, UNIQUE는 활성 행 한정 부분 인덱스.
- 주문 데이터는 **스냅샷** 보존(상품 변경에도 과거 주문 불변).
- 상세 ERD·테이블 분류·audit 정책은 [개요서 §6](MICOZ_프로그램개요서.md) 참조.
